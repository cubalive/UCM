PHASE 1 — DRIVER ROUTE (PICKUP vs DROPOFF)
---------------------------------------
1) Determine destination by trip status:
- If status in (SCHEDULED, ASSIGNED, EN_ROUTE_TO_PICKUP, ARRIVED_PICKUP): destination = pickup lat/lng (fallback pickup_address)
- If status in (PICKED_UP, EN_ROUTE_TO_DROPOFF, ARRIVED_DROPOFF, IN_PROGRESS): destination = dropoff lat/lng (fallback dropoff_address)

2) Origin:
- driver current GPS position (from navigator.geolocation)
- fallback to drivers.last_lat/lng if available

3) Use Google Directions API to compute route:
Option A (preferred for simplicity): use client-side DirectionsService:
- new google.maps.DirectionsService().route({origin, destination, travelMode:'DRIVING'})
- render using DirectionsRenderer OR manually draw polyline from overview_polyline

4) Persist route on trips:
- route_polyline = overview_polyline (encoded)
- duration_minutes, distance_miles, last_eta_minutes, last_eta_updated_at

Add/extend endpoint:
POST /api/trips/:id/route/recompute
Auth: assigned driver OR dispatch OR owning clinic (clinic only for its trips)
Input: origin lat/lng
Server calls Google Directions (or accept client response if key is client-restricted) and persists fields.
(Choose one approach and implement consistently.)

---------------------------------------
PHASE 2 — AUTO-REROUTE (THROTTLED)
---------------------------------------
Recompute route when ANY:
- moved >= 120 meters since last compute
- OR 20 seconds elapsed since last compute
- OR trip status changed (pickup->dropoff)
Throttle strictly to avoid spam + billing.

If driver is logged out OR not on active trip:
- stop recompute loop and stop location watch.

  - If using polyline: reuse same google.maps.Polyline and setPath(decodedPath)

Keep previous route visible while fetching a new one; replace when new data arrives.
