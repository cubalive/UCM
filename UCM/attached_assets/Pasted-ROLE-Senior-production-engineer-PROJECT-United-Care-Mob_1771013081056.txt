B) BACKEND: ADMIN CRUD ENDPOINTS (SUPER_ADMIN ONLY)
------------------------------------------------------------

1) Clinics
- PUT /api/admin/clinics/:id
  Editable:
    name, contact_name, contact_phone, contact_email,
    address fields (street/city/state/zip/lat/lng),
    city_id, notes, is_active
  Enforce city lock: clinic address city/state MUST match selected city_id.

- PATCH /api/admin/clinics/:id/archive
  Set is_active=false, deleted_at=now()
  Guardrail: block if clinic has ACTIVE trips (not completed/canceled) -> 409

- PATCH /api/admin/clinics/:id/restore
  Set is_active=true, deleted_at=null

- DELETE /api/admin/clinics/:id/permanent
  Allowed only if is_active=false
  Guardrails:
    - block if active trips exist -> 409
    - do NOT delete trips; preserve trip history
  Log to audit_log

2) Drivers
- PUT /api/admin/drivers/:id
  Editable:
    name, phone, email, license_number (force uppercase),
    city_id, vehicle_id (if applicable), notes, is_active

- PATCH /api/admin/drivers/:id/archive
  Guardrail: block if driver has ACTIVE trips assigned -> 409
  Set is_active=false, deleted_at=now()

- PATCH /api/admin/drivers/:id/restore
  Set is_active=true, deleted_at=null

- DELETE /api/admin/drivers/:id/permanent
  Allowed only if is_active=false
  Guardrail: block if active trips exist -> 409
  Preserve trips; remove only driver record and dependent non-trip rows if safe
  Log to audit_log

3) Patients
- PUT /api/admin/patients/:id
  Editable:
    name, phone, email(optional),
    address fields (street/city/state/zip/lat/lng),
    clinic_id, city_id, notes, is_active

- PATCH /api/admin/patients/:id/archive
  Guardrail: block if patient has ACTIVE trips -> 409
  Set is_active=false, deleted_at=now()

- PATCH /api/admin/patients/:id/restore
  Set is_active=true, deleted_at=null

- DELETE /api/admin/patients/:id/permanent
  Allowed only if is_active=false
  Guardrail: block if active trips exist -> 409
  Preserve trips; remove only patient record and safe dependent rows
  Log to audit_log

4) Dispatch Accounts (internal users)
Assume internal users are represented in profiles table with role='dispatch'.

- PUT /api/admin/users/:id
  Editable:
    name, email, role (dispatch/clinic_user/driver only), city_id (if used), is_active
  SUPER_ADMIN only.

- PATCH /api/admin/users/:id/archive
  Set is_active=false, deleted_at=now()
  Also disable sign-in:
    - set profiles.is_active=false and mark auth user disabled if supported,
      or block in backend middleware if profiles.is_active=false

- PATCH /api/admin/users/:id/restore
  Re-enable.

- DELETE /api/admin/users/:id/permanent
  Allowed only if is_active=false
  Must warn irreversible
  Log to audit_log
