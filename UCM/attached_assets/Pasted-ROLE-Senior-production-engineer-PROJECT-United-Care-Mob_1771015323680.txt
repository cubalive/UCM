- Supabase SMTP handles Supabase Auth system emails (already configured in Supabase)
- Resend API handles application transactional emails (login links, invites)

REQUIREMENTS:
1) Implement a server-side email sender using Resend
- Add a backend module/service email.ts (or similar) that sends emails with Resend.
- NEVER call Resend from frontend.
- Return clear JSON errors if token missing, from missing, invalid recipient, or Resend fails.

2) One-time magic link tokens
- Create DB table if missing: login_tokens
  columns:
    id (uuid pk)
    token_hash (text unique)
    user_id (uuid nullable)
    clinic_id (text nullable)
    driver_id (text nullable)
    role (text)
    expires_at (timestamptz)
    used_at (timestamptz nullable)
    created_by (uuid nullable)
    created_at (timestamptz default now())
- Hash token with SHA-256 before storing (store only hash). Never store raw token.
- Token TTL: 15 minutes. One-time use.

3) Backend endpoints
A) POST /api/admin/send-login-link
- Auth required: super_admin only (server-side enforced)
- Input: { targetType: "clinic"|"driver", targetId: string }
- Look up target record -> must have email.
- Create raw token, hash it, insert into login_tokens with role and target references.
- Build link: `${APP_PUBLIC_URL}/login?token=${rawToken}`
- Send email via Resend including link.
- Audit log record: action, target, status, timestamp.
- Rate limit: max 1 per recipient per 60 seconds.

B) POST /api/auth/token-login
- Input: { token: string }
- Hash token; find login_tokens where token_hash matches AND expires_at>now AND used_at is null
- Mark used_at = now
- Sign in user:
   Option 1 (preferred): use Supabase Admin (service_role) to create a Supabase Auth session for that user if system supports it
   Option 2: if app already uses JWT mode, mint app JWT and set cookie/session consistent with existing auth
- Return success + redirect info.

4) Frontend wiring
- Hook “Send Clinic Login Link” button to call POST /api/admin/send-login-link
- Show proper toasts: missing email disables button; success shows “Login link sent”; errors show reason.
- Update /login page:
   If token query param exists, call POST /api/auth/token-login then route user to their dashboard.

5) Diagnostics

  { hasResendKey: boolean, from: string, appPublicUrl: string }
