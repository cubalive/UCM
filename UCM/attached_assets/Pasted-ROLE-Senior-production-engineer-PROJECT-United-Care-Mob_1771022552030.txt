GOAL:

2) Implement fast vehicle reassignment / driver swap for same-day failures:
   - Dispatch + Super Admin can reassign vehicles and swap drivers safely without breaking trips.
   - Keep audit trail.

PERMISSIONS (server-side enforced):
A) SUPER_ADMIN
- Can archive/restore/hard-delete vehicles and drivers (hard-delete only from Archived)
- Can reassign vehicles and swap drivers

B) DISPATCH
- Can archive vehicles and drivers (soft delete only)
- Can reassign vehicles and swap drivers
- Cannot hard-delete

C) CLINIC_USER
- No access to vehicles/drivers deletion
- Only sees assigned drivers for their trips as already defined

DATA MODEL (add if missing):
1) Soft delete fields on vehicles and drivers:
- deleted_at timestamptz null

CORE FEATURES:

PHASE 1 — ARCHIVE VEHICLES & DRIVERS
- Add DELETE /api/vehicles/:id
  allowed: super_admin or dispatch
  action: soft delete (set deleted_at, deleted_by)
- Add POST /api/vehicles/:id/restore (super_admin only)
- Add POST /api/vehicles/:id/hard-delete (super_admin only, archived only)
- Add DELETE /api/drivers/:id
  allowed: super_admin or dispatch
  action: soft delete
- Add restore + hard-delete for drivers (super_admin only)

Rules:
- If a vehicle/driver has active future assignments or active trips:
  - Do not hard delete
  - For soft delete, require a reason and block new assignments
  - Return clear warning list of affected assignments/trips

PHASE 2 — QUICK REASSIGN / SWAP (DISPATCH + ADMIN)
A) Reassign a vehicle for a driver for today
- Endpoint: POST /api/dispatch/assignments/reassign-vehicle
  body: { city_id, shift_date, driver_id, from_vehicle_id, to_vehicle_id, apply_to_trips: boolean }
  rules:
  - validate driver and vehicle belong to same city_id
  - ensure to_vehicle is not already assigned for that shift_date (or handle override with warning + confirm)
  - update driver_vehicle_assignments row: set vehicle_id = to_vehicle_id, updated_by, notes
  - if apply_to_trips=true:
     update all trips for that driver on shift_date that are not completed/cancelled to use new vehicle_id

B) Swap drivers (driver A fails; driver B covers)
- Endpoint: POST /api/dispatch/assignments/swap-driver
  body: { city_id, shift_date, from_driver_id, to_driver_id, apply_to_trips: boolean, keep_vehicle: 'from'|'to'|'manual', manual_vehicle_id?: id }
  rules:
  - validate both drivers same city_id and available (not deleted, not on hold, not already maxed)
  - update driver_vehicle_assignments:
     move assignment from from_driver -> to_driver (status='reassigned', audit)
  - if apply_to_trips=true:
     update trips scheduled that day:
       trips.driver_id = to_driver_id
       vehicle_id depends on keep_vehicle setting
  - log audit record of swap
