PHASE 2 — IMPLEMENT EXACTLY THESE 10 INDEXES
------------------------------------------------------------

INDEX #1: Transport Reliability Index Nevada™ (TRI) 0–100
Uses:
scheduled_pickup_at, arrived_pickup_at, completed_at, billing_outcome/status
Definitions:
OnTime = arrived_pickup_at <= scheduled_pickup_at + grace_minutes
Late   = arrived_pickup_at  > scheduled_pickup_at + grace_minutes
NoShow = outcome/status no_show
TRI = 100*(OnTimeCompleted/Completed) - late_weight*100*(LateCompleted/Completed) - noshow_weight*100*(NoShow/(Completed+NoShow))
Return: score + counts (completed, ontime, late, noshow)

INDEX #2: Clinic Trust Score (CTS) 0–100
Composite (weights normalized if some components missing):
- TRI (normalized)
- Return Reliability (on-time returns %)
- Proof Completeness (timestamp completeness + proof photo presence %)
- Dispute Rate (if tracked) else omit
Return: score + component breakdown

INDEX #3: Driver Stability Index 0–100
Stability = completed/assigned
Penalize late pickups and driver-attributed cancels if data exists
Return: score + counts

INDEX #4: Driver Utilization Index %
Utilization = active_trip_minutes / scheduled_minutes
active_trip_minutes = picked_up_at→completed_at (fallback arrived_pickup_at→completed_at)
Return: percent + minutes

INDEX #5: Dispatch Efficiency Score %
Efficiency = auto_assigned / total_assigned
Track manual overrides/reassignments via existing flags or status history inference
Return: efficiency% + manual_override_count + reassignment_count

INDEX #6: Revenue Leakage Index ($)
Leakage = sum of:
- no_show not billed
- cancelled billable but not billed
- return legs missing billing record when trip exists
- private completed unpaid (if private module exists)
Return: leakage_total + leakage_by_reason + leakage_by_clinic

INDEX #7: Clinic Load Index (ratio + level)
Load = active_trips / scheduled_drivers
Return: ratio + level (low/med/high) + thresholds used

INDEX #8: Weekly Profit Index ($ + margin%)
Profit = revenue - driver_cost - vehicle_cost_estimate
Revenue source: clinic billing runs/invoices preferred; else sum billed amounts
Costs: use existing pay model; else configurable estimates
Return: revenue$, cost$, profit$, margin%

INDEX #9: Replacement Pressure Index (risk)
Pressure = scheduled_drivers - active_drivers
active_drivers = online OR assigned trips
Return: shortage_count, risk level, recommended action

INDEX #10: Late Risk Predictor (0–100 risk + reasons)
Light heuristic (no external APIs):
Risk factors:
- historical lateness for same city/time bucket
- driver schedule buffer tightness
- clinic load high
- driver stability low
- long distance vs time to appointment
Return:
- summary: red/yellow/green counts
- top 20 at-risk trips with reason list
