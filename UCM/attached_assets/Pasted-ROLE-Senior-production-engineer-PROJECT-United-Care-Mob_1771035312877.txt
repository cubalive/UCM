PHASE 2 — RULE ENGINE (CITY + ZIP CLUSTERING)
-----------------------------------------------------
Implement a backend service function:
generateAssignmentPlan({runAt, cities:["Las Vegas","Pahrump"]})

Rules:
1) Hard rule: NEVER mix cities in grouping or assignment.
2) Within each city:
   - group trips by pickup_zip (fallback: derive zip from pickup address if we store it; if unavailable, group as "unknown_zip")
   - prioritize trip_type order:
       dialysis first
       recurring second
       one_time last
3) Only include trips that are:
   - status in ("pending","approved") AND not cancelled/completed
   - if clinic-created trips: they can be "pending" but must NOT be assigned unless approved_by_dispatch=true
4) Driver eligibility:
   - driver.home_city must match city
   - driver not on hold
5) Vehicle eligibility:
   - vehicle.home_city matches city
   - vehicle.active=true
   - vehicle_type must match trip needs (wheelchair trips require wheelchair vehicle; sedan trips cannot use wheelchair-only if you have such flag; if wheelchair vehicle can do sedan trips allow it via config)
6) Preference:
   - keep same patient->driver pairing if we have last_assigned_driver_id or prior trip history
   - keep driver->vehicle pairing if available
7) Output:
   - create assignment_batch row per city
   - write proposed assignments onto trips (assigned_driver_id/vehicle_id) ONLY IF "apply" action confirmed, otherwise store proposals in memory or a proposals table.
   - include assignment_reason text for transparency.
