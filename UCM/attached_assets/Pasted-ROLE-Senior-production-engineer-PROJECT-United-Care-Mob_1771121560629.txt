========================================================
CORE OPERATIONAL RULES
========================================================

1) Weekly schedule exists Monday–Saturday.
2) Sunday is OFF by default.
3) Dispatch can enable Sunday and manually add drivers.
4) Each day can also have SUBSTITUTE DRIVERS ("suplentes").
5) Substitutes are NOT part of the main schedule unless used to replace someone.
6) If a scheduled driver cannot work:
   - If there is time: dispatch edits schedule and swaps driver.
   - If shift already started: dispatch marks driver OUT and assigns a substitute.
7) When pressing "REASSIGN ROUTES":
   - ONLY trips for the OUT driver that have NOT STARTED move to substitute.
   - Trips already started (en route, picked up, arrived) remain untouched.
   - Other drivers remain untouched.
8) Logged-out drivers must NEVER appear in "Unassigned Drivers".
9) Only drivers BOTH:
   - scheduled for that date
   - AND logged in
   appear in Unassigned.
10) Substitutes can be used only if:
   - in substitute pool for that date
   - logged in

PHASE 5 — DAILY ASSIGNMENTS LOGIC
========================================================

For selected (city, date):

scheduledDrivers =
Mon–Sat → weekly schedule enabled
Sunday → sunday_roster if enabled

eligibleDrivers =
scheduledDrivers WHERE logged in

driversWithTrips =
distinct drivers assigned that day

unassignedDrivers =
eligibleDrivers MINUS driversWithTrips

UI must show:
Unassigned Drivers = unassignedDrivers ONLY

Logged-out drivers must never appear.

Substitutes must not appear in Unassigned by default.

========================================================
PHASE 6 — CONTROLLED REASSIGN SYSTEM
========================================================

Add button:
"REASSIGN AFFECTED ROUTES"

When pressed:

For each replacement mapping:
(out_driver → substitute_driver)

Find trips for that date where:
trip.driver_id = out_driver
AND trip.status IN:
- scheduled
- assigned
- pending

DO NOT move trips where status IN:
- en_route
- arrived
- picked_up
- completed
- cancelled

Reassign eligible trips:
trip.driver_id = substitute_driver

Do NOT touch:
- other drivers
- already started trips

If substitute logged out:
BLOCK and show:
"Substitute must be logged in."
