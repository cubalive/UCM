---------------------------------------

  EN_ROUTE_TO_PICKUP -> "Arrived Pickup"
  ARRIVED_PICKUP     -> "Picked Up"
  PICKED_UP/IN_TRANSIT -> "Arrived Dropoff"
  ARRIVED_DROPOFF    -> "Complete Trip"

- Prevent skipping steps; server enforces valid transitions.

PHASE C — OFFLINE-FIRST (driver never gets stuck)
---------------------------------------
Add an offline queue (localStorage/IndexedDB):
- Queue items:
  - status transitions
  - support events
  - GPS updates (only keep last N; don’t explode storage)
- When offline:
  - UI shows "Offline — saving changes"
  - actions write to queue and update UI optimistically
- On reconnect:
  - flush queue in order with backoff
  - show “Synced” toast

---------------------------------------
