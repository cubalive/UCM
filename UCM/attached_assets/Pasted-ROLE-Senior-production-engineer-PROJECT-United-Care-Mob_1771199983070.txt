---------------------------------------
Implement WebSocket channel per trip:

channel: trip:{tripId}

When clinic opens trip:
- send snapshot
- subscribe to channel

Events:
- driver_location (max every 5s)
- status_change (instant)
- ETA_update (max every 60s)

Disconnect when page closes.

---------------------------------------
PHASE 3 — DRIVER LOCATION INGEST
---------------------------------------
Create endpoint:

POST /api/driver/location

Accept:
- single point OR batch

Validate:
- reject updates faster than 1 per 2s
- reject timestamp older than 60s
- reject impossible jumps

Do NOT write every update to Postgres.

Store last location in memory cache with TTL 120s.

---------------------------------------
PHASE 4 — CACHE LAYER
---------------------------------------
Implement in-memory cache for now.

Structure:
driver:{id}:last_location
trip:{id}:driver_last
trip:{id}:eta

TTL:
driver last: 120s
ETA: 60s

Must be swappable to Redis later.

---------------------------------------
PHASE 5 — ETA THROTTLE
---------------------------------------
Google Directions must NOT run on every update.

Recompute ETA only if:
- driver moved >300m
OR
- 45s passed

Cache result 60s.

Fallback:
haversine distance.
