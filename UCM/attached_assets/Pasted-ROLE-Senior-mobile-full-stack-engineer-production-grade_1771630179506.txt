A) DEFINE A SINGLE SOURCE OF TRUTH: Trip Status Model

   - SCHEDULED
   - EN_ROUTE_TO_PICKUP
   - ARRIVED_PICKUP
   - PICKED_UP
   - EN_ROUTE_TO_DROPOFF (or IN_PROGRESS)
   - ARRIVED_DROPOFF
   - COMPLETED
   - CANCELED (if exists)
2) Create a small pure function:
   getTripPhase(trip, now) -> phase

   - If backend stores timestamps (arrived_pickup_at, picked_up_at, started_at, arrived_dropoff_at, completed_at), derive phase from timestamps.
   - Never guess dropoff actions before pickup is completed.

B) UI BUTTONS MUST ALWAYS MATCH PHASE
On Trip Detail, show action buttons exactly as follows:

PHASE: SCHEDULED
- Primary: "Go to Pickup" (opens nav to pickup)
- Secondary: "Mark Arrived at Pickup" (enabled only if within geofence OR always allowed with confirm)
- Show pickup and dropoff addresses; show timeline dots.

PHASE: EN_ROUTE_TO_PICKUP
- Primary: "Go to Pickup"
- Secondary: "Mark Arrived at Pickup"

PHASE: ARRIVED_PICKUP
- Primary: "Picked Up"
- Secondary: "Go to Pickup" (still allowed)
- Optional: show waiting timer policy if exists (not required)

PHASE: PICKED_UP
- Primary: "Start Trip" (sets status to EN_ROUTE_TO_DROPOFF / IN_PROGRESS)
- Secondary: "Go to Dropoff" visible but disabled until Start Trip if you prefer; OR allow both but Start Trip must be available.

PHASE: EN_ROUTE_TO_DROPOFF / IN_PROGRESS
- Primary: "Go to Dropoff"
- Secondary: "Mark Arrived at Dropoff"

PHASE: ARRIVED_DROPOFF
- Primary: "Complete Trip"
- Secondary: "Go to Dropoff"

PHASE: COMPLETED
- No action buttons except "View Receipt/Details" if exists.

Rules:
- Never show "Go to Dropoff" as the only action while still SCHEDULED/EN_ROUTE_TO_PICKUP.
- Never allow "Complete" before Arrived Dropoff.
- Each button must call one clear API/mutation and update UI immediately (optimistic update ok).
