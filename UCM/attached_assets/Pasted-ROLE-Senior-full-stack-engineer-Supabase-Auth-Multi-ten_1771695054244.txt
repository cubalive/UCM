https://clinic.unitedcaremobility.com/

2) Keep other roles redirecting to the correct portal (app., dispatch., admin. if used)
3) Add a single source of truth function that decides redirect base by role
4) Add logs and a small test to prevent regression

REQUIRED ENV VARS
- APP_BASE_URL=https://app.unitedcaremobility.com
- CLINIC_APP_URL=https://clinic.unitedcaremobility.com
(Optional) DISPATCH_APP_URL, ADMIN_APP_URL

TASKS

Search for:
- inviteUserByEmail
- generateLink
- magiclink
- redirectTo / redirect_to
- /auth/v1/verify
- "type=magiclink"

B) Implement helper:
getPortalBaseUrlByRole(role: string): string
Rules:
- role === 'CLINIC' -> CLINIC_APP_URL
- role === 'SUPER_ADMIN' -> APP_BASE_URL (or ADMIN_APP_URL if present)
- role === 'DISPATCH' -> APP_BASE_URL (or DISPATCH_APP_URL)
- default -> APP_BASE_URL

If using generateLink:
supabase.auth.admin.generateLink({
  type: 'magiclink' or 'invite',
  email,
  options: { redirectTo: `${baseUrl}/` }
})
