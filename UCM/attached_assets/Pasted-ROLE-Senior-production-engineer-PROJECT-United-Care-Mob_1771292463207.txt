TASK (PHASE 3 — CLINIC TRIP DETAILS + PDF EXPORT):
Goal: Clinics need a dedicated Trip Details view/modal that shows ONLY informational trip data (no action buttons, no notes/messages UI, no patient link actions). It must include a clean A→B summary with map snapshot and a full timeline of timestamps. It must be exportable/downloadable as PDF.

CURRENT PROBLEM:
- Clicking "Trip completed" (clinic view) does not show complete trip details.
- Existing trip progress is missing fields and the clinic needs a “Trip Details” designed for them.

REQUIREMENTS:
1) Create a ClinicTripDetails component/page/modal used in clinic role only.
   - Must NOT include driver action buttons, messaging UI, internal notes, or editable controls (clinic can’t change trip except via dispatcher flow).
   - Must show: Patient name (display-only), Trip ID, Clinic name, pickup address, dropoff address, city, service type (sedan/wheelchair), special needs flags, assigned driver + vehicle (if assigned).

2) Add full timeline fields (read-only):

   - scheduledDropoffTime (or estimated duration + ETA if dropoff time not stored)

     - Use Google Static Maps API OR existing map snapshot approach if already present.
     - Include markers A and B and a polyline if available.
   - Must not require live map interaction for clinic.

   - Use authenticated download approach (reuse downloadWithAuth from PHASE 2):

     - Header: United Care Mobility + Clinic name + Trip ID
     - Map snapshot
     - A→B addresses
     - Status + timeline table
     - Driver/vehicle (if assigned)
