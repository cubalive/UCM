PHASE 2 — BACKEND API (Express)
========================================================
Implement these endpoints with clean validation and safe auth checks:

1) GET /api/map/bootstrap?city=las-vegas
Return:
- active drivers with last location + driver name
- joined vehicle info (vehicle name + color_hex)
- active trips for today with status + eta_minutes

2) POST /api/drivers/location
Body: { lat, lng }
Auth: driver session required (or token)
Store driver last_lat/last_lng/last_seen_at.

3) GET /api/trips/:id/eta
Compute ETA + distance/time using Google Directions API.
Store results into trips: eta_minutes, distance_miles, duration_minutes, route_polyline, updated_at.

4) POST /api/trips/:id/recalc
Auth: dispatch or super_admin
Recalculate route metrics for that trip using Google Directions.
Return updated values.

5) GET /api/clinic/trips?city=las-vegas
Auth: clinic_user (or public read if you haven’t built clinic auth yet — if not built, keep it behind auth for now)
Return trips with: status + eta_minutes + “arriving in X min” string.
