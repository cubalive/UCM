REQUIREMENTS (MUST IMPLEMENT EXACTLY)
A) PATIENT NOTIFICATIONS (TWILIO SMS)
1) 24-hour reminder:
- When a trip is scheduled >= 24 hours in advance, send SMS exactly 24 hours before scheduled_pickup_at.
- Message contains: pickup time, pickup address, dropoff address, vehicle info (if assigned), driver name (if assigned), tracking link (if available), and "reply HELP" instructions.
- Must be idempotent: never send twice for the same trip notification type.

2) Short-notice trips (<24 hours):
- Do NOT wait for 24-hour reminder.
- If scheduled_pickup_at is within next 24 hours, send a "Trip scheduled" SMS immediately upon trip creation (or upon patient being attached), including same info and tracking link.

3) "Driver on the way" (en-route) alert:
- When driver presses "On My Way" (status = en_route_pickup), send an SMS immediately:
  "Your driver is on the way. ETA: X minutes. Track here: <link>"

4) "Arrived at pickup" alert:
- When driver reaches geofence and confirms arrival (status = arrived_pickup), send SMS:
  "Your driver has arrived."

5) "Trip started" alert:
- When driver presses Start Trip (status = in_progress), send SMS:
  "Your trip has started."

6) "Trip completed" alert:
- When driver presses Finish Trip (status = completed), send SMS:
  "Your trip is complete. Thank you. Rate your ride: <link>"

B) DRIVER APP BUTTON WORKFLOW (STATE MACHINE)
Driver UI must show ONLY one primary action button at a time, based on current trip status:

1) Button: "On My Way"
- Visible when trip is assigned and status = scheduled/assigned
- On press -> backend transition to status = en_route_pickup
- Must record timestamp: en_route_pickup_at

2) Button: "Arrived at Pickup"
- Visible ONLY when status = en_route_pickup
- Enabled ONLY when driver location is within pickup geofence radius (default 150m, configurable per company)
- On press -> transition to status = arrived_pickup
- Record: arrived_pickup_at
- If not within geofence, show clear message: "You must be near the pickup location to mark arrival."

3) Button: "Start Trip"
- Visible ONLY when status = arrived_pickup
- On press -> transition to status = in_progress
- Record: trip_started_at

4) Button: "Finish Trip"
- Visible ONLY when status = in_progress
- Enabled ONLY when driver location is within dropoff geofence radius (default 150m, configurable per company)
- On press -> transition to status = completed
- Record: completed_at

C) NOTES / ANNOTATIONS
- Driver must have an optional notes field at completion (and/or anytime):
  - "Add note" opens text area
  - Notes saved to trips.driver_notes (append-only log with timestamp + driver_id)
- Dispatch/super_admin can view notes.
- Notes are included in audit trail.

D) GEOFENCE + LOCATION PINGS
- Driver app must send location pings every 15–30 seconds while:
  - status in (en_route_pickup, arrived_pickup, in_progress)
- Store to gps_pings table: trip_id, driver_id, lat, lng, ts, company_id
- Geofence check uses the latest ping (or current reading) to validate distance.

E) SERVER-SIDE ENFORCEMENT
- Implement a strict status state machine on the server:
  assigned/scheduled -> en_route_pickup -> arrived_pickup -> in_progress -> completed
- Reject invalid transitions with JSON error:
  { ok:false, code:"INVALID_TRANSITION", error:"..." }

- Implement geofence checks server-side for arrival + completion:
  If outside radius -> reject with { ok:false, code:"GEOFENCE_REQUIRED", error:"..." }
