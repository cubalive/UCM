GOALS (must be true)
1) Trip lifecycle categories:
   - SCHEDULED: trip date is today, not started (SCHEDULED or ASSIGNED)
   - ACTIVE_NOW: EN_ROUTE_TO_PICKUP, ARRIVED_PICKUP, ON_TRIP, EN_ROUTE_TO_DROPOFF, ARRIVED_DROPOFF
   - COMPLETED/CANCELLED/NO_SHOW terminal
2) Driver states:
   - AVAILABLE: can accept new offers
   - RESERVED: has accepted a future trip but depart_at not reached; still can accept other trips IF no conflict
   - BUSY: only when trip is active (en route / waiting / on trip)
3) depart_at calculation:
   - depart_at = pickup_time - (ETA(driver->pickup) + buffer)
   - buffer default 5 min (config)
   - If driver is far, depart_at moves earlier automatically.
4) Notification logic:
   - When driver accepts future trip: show “Reserved” card, not “Active trip”.
   - At (depart_at - 5 min): push alert “Get ready”
   - At depart_at: alert “Leave now” + optionally auto-switch trip to EN_ROUTE_TO_PICKUP if driver confirms “Start”
5) Conflict checking:
   - When offering/auto-assigning, ensure no overlap:
     previousTripEnd + travelTimeToNextPickup <= nextPickupTime
   - Add minimum buffer between trips (ex 10 min) configurable.
