C) DRIVER PORTAL: PROFESSIONAL OPERATION MODE
========================================================
IMPORTANT: NO “Always” location. Only “When In Use”. Track only while the app is open.

1) SHIFT MODE
- Add Start Shift / End Shift on Home.
- While shift is ON:
  - driver status becomes “Available/Online”
  - can accept trips
- While shift is OFF:
  - driver status becomes “Offline”
  - cannot accept new trips
- Persist shift sessions in DB:
  - id, driverId, companyId, startAt, endAt, breaks[], totalMinutes, createdAt
- Add Break toggle (Start break/End break).

2) TRIP CENTER (single place to work)
- For any active/assigned trip, show Trip Center with:
  - pickup time, ETA, countdown
  - addresses, patient name (if allowed), clinic name, notes
  - action buttons in sequence:
    En route -> Arrived -> Picked up -> Start Trip -> Drop off -> Complete
- Each action writes timestamp fields + audit log row:
  - enRouteAt, arrivedAt, pickupAt, tripStartAt, dropoffAt, completedAt
- Add “Open Navigation” (Apple/Google maps deep link).

3) IN-APP GEOFENCE (foreground only)
- While Trip Center is open, poll current position (foreground).
- Compute distance to pickup and dropoff.
- Configurable radius (default 150m) per company:
  - If within radius of pickup: show “You’re at pickup” + enable Arrived.
  - If within radius of dropoff: enable Drop off.
- If location denied:
  - show “Limited mode: enable location to unlock auto-arrival”
  - still allow manual Arrived with confirmation modal.

4) NO-SHOW PROTOCOL
- When driver taps Arrived:
  - start a configurable wait timer (5/10/15 minutes per company)
  - show buttons: Call patient, Text patient, Notify dispatch
  - log attempts as evidence
- Mark No-Show:
  - allowed only after timer reaches 0 OR if override permission with reason
  - capture reason + evidence (at least one attempt or note)

5) DIGITAL SIGNATURE / PROOF OF SERVICE
- At Picked up and/or Drop off (company config):
  - signature pad capture
  - “patient refused to sign” option + reason
- Store signature artifact (image) + link on trip record.
- Show signature status on Trip Center and Trip Details.

6) HOURLY PAYROLL + REAL-TIME EARNINGS
- Hourly payroll is driven by Shift Sessions + Breaks.
- Generate timecards automatically:
  - shift start/end -> time entries
  - allow admin/dispatch edits with audit logs
- Driver Earnings tab:
  - Today / Week / Month
  - breakdown: hours * rate, bonuses, adjustments
- Admin Payroll:
  - Weekly payroll run aggregates hours and computes gross pay
  - export CSV (optional) without breaking anything
