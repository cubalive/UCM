PHASE 2 — BACKEND (AUTHORITATIVE ROUTE ENGINE)
Implement a “Route Engine” module with ONE source of truth:
- It produces a polyline + distance/time for (a) planned route and (b) actual driven route.

2.1 Location ingest (already exists) — enhance without breaking:
- Ensure driver location ping includes trip_id when on active trip.
- Store every ping (throttled): accept 1 point every 5 seconds OR if moved > 20 meters OR heading change > 25°.
- Insert into trip_location_points with tenant_id/city_id/driver_id from auth context.
- Add idempotency: reject duplicates for same trip_id + ts.

2.2 Planned route (for admin/dispatch/clinic view):
- On trip creation or assignment, call Google Routes API from SERVER using:
  origin: pickup place_id if available else lat/lng
  destination: dropoff place_id if available else lat/lng
  travelMode: DRIVE
  routingPreference: TRAFFIC_AWARE
  requested fields: polyline.encodedPolyline, distanceMeters, duration
- Save to trips.route_polyline / route_distance_meters / route_duration_seconds / route_source='routes_api'
- IMPORTANT: This must be server-side only.

2.3 Actual route (Uber-like playback):
- When trip is completed, compute actual route:
  Option A (preferred): Use telemetry points -> Snap to Roads (if enabled) -> build polyline -> compute distance
  Option B: If Roads API not available, compute haversine distance from points with smoothing + outlier removal.
- Save to trips.route_polyline (actual) and actual_distance_meters, actual_duration_seconds, route_source='telemetry', route_quality_score.
- Keep planned route separately if you already store it; if not, add columns planned_polyline/planned_distance/planned_duration OR store planned in new table trip_route_versions.

2.4 Waiting timer + arrived gating
- Use geofence check using pickup/dropoff coordinates or place_id geometry:
  When driver within pickup_geofence_m AND status transitions to ARRIVED_PICKUP:
    set arrived_pickup_at = now()
  Waiting begins at arrived_pickup_at and ends at picked_up_at
  waiting_seconds = EXTRACT(EPOCH FROM (picked_up_at - arrived_pickup_at))
- Similar for dropoff.

2.5 Endpoints (do not break existing routes)
Add:
GET /api/trips/:tripId/route
- returns { polyline, distanceMeters, durationSeconds, source, quality, updatedAt }
- obeys RBAC/tenant/city
GET /api/trips/:tripId/playback?from=...&to=...
- returns compressed points or polyline segments
POST /api/internal/trips/:tripId/finalize
- computes route + miles + waiting + timeline and persists (call on completion)
Make sure static routes are declared before param routes as already learned.
