PHASE 1 — DRIVER ↔ VEHICLE LINK (REQUIRED)
---------------------------------------------------

Ensure:

drivers.vehicle_id exists  
vehicles.city_id exists  
drivers.city_id exists  

Rules:
- A driver can only use ONE vehicle at a time
- A vehicle can only be assigned to ONE driver at a time
- Driver + vehicle must belong to same city

Create endpoint:

POST /api/dispatch/assign-driver-vehicle

Body:
{
  driver_id,
  vehicle_id
}

Logic:
- verify both exist
- verify same city
- verify vehicle not in use by another active driver
- update driver.vehicle_id
- return updated driver + vehicle

PHASE 2 — TRIP ASSIGNMENT ENGINE
---------------------------------------------------

Create endpoint:

POST /api/dispatch/assign-trip

Body:
{
  trip_id,
  driver_id
}

Logic:
1) verify trip exists
2) verify driver exists and active
3) driver must have vehicle assigned
4) trip.city_id must match driver.city_id
5) assign:

trip.assigned_driver_id
trip.assigned_vehicle_id
trip.status = 'assigned'

6) immediately calculate ETA using Google Directions
7) store:
eta_minutes
distance_miles
duration_minutes

Return full trip object.

POST /api/dispatch/auto-assign

{

}

Logic:
1) get all unassigned trips in city
