- created_at timestamptz default now()

PHASE D — ENDPOINTS
==================================================

Implement endpoints:

1) POST /api/admin/imports
Body: { companyId, cityId?, sourceSystem, consentConfirmed:true }
-> create import_job

2) POST /api/admin/imports/:id/upload?entity=...
- Accept multipart file upload (CSV/XLSX/JSON)
- Store into import_job_files.storage_json
- Return file metadata

3) POST /api/admin/imports/:id/validate
- For each uploaded entity file:
  - parse
  - map to canonical
  - validate required fields
  - dedupe in-file
  - check duplicates vs DB using external_id_map and natural keys (email/phone/license_plate)
- Must NOT write to main tables
- Save summary_json + import_job_events (errors/warnings)
Return:
{ status:'validated', counts:{ok,error,skipped}, preview:{entity:[first20]}, errorsDownloadToken }

4) POST /api/admin/imports/:id/run
- Must require job status validated
- Execute in safe order:
  a) clinics
  b) patients
  c) drivers
  d) vehicles
- Upsert logic:
  - If row has external_id and exists in external_id_map -> update existing (safe fields only)
  - Else attempt match by natural key:
     clinics: name + city + phone/email
     patients: (first+last+dob) OR phone; plus clinic match
     drivers: phone or email
     vehicles: license_plate
  - If match found -> update + optionally backfill external_id_map
  - If no match -> insert + insert external_id_map
- Record per-row results in import_job_events
- Set job status completed/failed

5) POST /api/admin/imports/:id/rollback
- Rollback only rows created by this import job:
  - Use auditLog entries OR import_job_events payload to track inserted ids
- Must NOT delete any rows that pre-existed.
- Set status rolled_back

6) GET /api/admin/imports
7) GET /api/admin/imports/:id
