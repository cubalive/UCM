   - ETA minutes + distance
2) Auto-reroute:
   - Watch driver GPS updates.
   - Throttle route recompute to every 15–30 seconds OR when moved >= 100–150 meters OR bearing change is large.
   - If trip status changes (pickup -> dropoff), immediately recompute route.
3) Smooth update:
   - Do NOT clear the entire map on each refresh.
   - Update only the polyline layer + marker positions.
   - Avoid flicker by reusing map instance and renderer objects.
4) Navigation button:
   - Keep the “Choose app” modal (Google/Apple/Waze) and open external navigation.
   - Even if external nav is used, keep in-app route polyline updated.

3) If no active trips:
   - Show banner: “No live routes for your clinic right now.”

BACKEND / DATA:
1) Store computed route polyline + eta in trips:
   - route_polyline, duration_minutes, distance_miles, last_eta_minutes, last_eta_updated_at
2) Add endpoint:
   - POST /api/trips/:id/route/recompute
   - Auth: driver of trip OR dispatch OR clinic that owns trip (clinic only if trip belongs to them)
   - Input: origin lat/lng (driver position), target based on trip status
   - Output: polyline + eta + distance; persist to DB
3) Driver app calls recompute with throttle; Clinic can read persisted polyline/eta and optionally request recompute only if updated_at is stale > 60s.
