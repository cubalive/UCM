PHASE 2 — PRICING ENGINE (SERVER-SIDE)
---------------------------------------------------------
4) Implement function calculateTripPrice(trip, clinicProfile):
Inputs:
- trip.trip_type (one_way/round_trip) if exists, else infer from legs
- pickup/dropoff lat/lng if available (optional mileage)
- clinic pricing profile

Rules:
A) DIALYSIS_FLAT:
- one_way -> flat_one_way_cents
- round_trip -> flat_round_trip_cents (or 2x one_way if round not set)

B) CONTRACT_FLAT:
- same as flat (use flat_one_way_cents or base_fee_cents)

C) MILEAGE:
- amount = base_fee_cents + (per_mile_cents * estimated_miles)
- estimated_miles:
  - If we have distance from Directions API saved, use it
  - else approximate via haversine as fallback (do not break)
- apply minimum/maximum if present

D) PRIVATE:
- allow manual pricing only (do not auto-set) OR compute with MILEAGE rules (choose simplest)

Always output:
- price_cents
- breakdown JSON: {type, base_fee, miles, per_mile, minimum_applied, maximum_applied, computed_at}

5) When to compute:
- On dispatch approval of clinic-created trip (or when trip becomes APPROVED):
  - load clinic_pricing_profile
  - auto-calc and set trips.price_cents if profile active
  - set price_source='AUTO', priced_at, price_breakdown

6) Override:
- PATCH /api/trips/:id/price (dispatch/super_admin)
- If clinic profile allow_dispatch_override=true:
  - set price_source='MANUAL_OVERRIDE'
  - store priced_by, priced_at
  - update breakdown {override:true, previous_auto:..., reason?:...}
- If not allowed, return 403

7) Locking:
- When invoice is FINAL -> price_locked=true and block edits.
- If trip completes before pricing is set:
  - invoice remains DRAFT with status NEEDS_PRICING (or DRAFT + missing price)
  - dispatch/admin must set price then finalize invoice.
