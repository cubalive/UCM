---------------------------------------
TASK 4 — MULTI CSV EXPORTS (B)
---------------------------------------
Create 6 exporters, each building rows with explicit columns.

1) SUMMARY CSV
Filename: ucm-metrics-summary-YYYYMMDD-HHMM.csv
One row with KPIs:
- generatedAt
- healthStatus
- reqPerMin
- errorRatePct
- p95LatencyMs
- redisOk
- cacheHitRatePct
- realtimeTokensIssuedPerMin
- realtimePublishesPerMin_location
- realtimePublishesPerMin_eta
- realtimePublishesPerMin_status
- directionsCallsPerMin
- directionsFailuresPerMin
- breakerOn

If some fields are missing in your metrics JSON, map best-effort and leave empty strings.

2) HEALTH CSV
Filename: ucm-metrics-health-YYYYMMDD-HHMM.csv
One row with:
- generatedAt
- overall (GREEN/YELLOW/RED)
- db (ok/fail + latency if present)
- redis (ok/fail)
- gpsStaleRate (if present)
- lastError (if present)

3) REDIS/CACHE CSV
Filename: ucm-metrics-cache-YYYYMMDD-HHMM.csv
Rows for each cache family (or one row with multiple columns):
Preferred rows format:
- generatedAt
- keyFamily (trip:eta, trip:polyline, trip:driver_location, driver:last_location)
- hits
- misses
- hitRatePct

If only aggregate hit/miss exists, export one row with keyFamily="all".

4) REALTIME CSV
Filename: ucm-metrics-realtime-YYYYMMDD-HHMM.csv
Rows per event type:
- generatedAt
- eventType (driver_location / eta_update / status_change / test_ping)
- publishesPerMin
Plus header-level fields (separate row with eventType="__totals__"):
- tokensIssuedPerMin
- lastPublishAt (if present)

5) GOOGLE CSV
Filename: ucm-metrics-google-YYYYMMDD-HHMM.csv
One row:
- generatedAt
- directionsCallsPerMin
- directionsFailuresPerMin
- breakerOn
- breakerRemainingSec (if present)
- lockContentionCount (if present)
- timeoutsCount (if present)

6) ROUTES CSV (optional)
Filename: ucm-metrics-routes-YYYYMMDD-HHMM.csv
Rows from /api/ops/metrics/routes:
- generatedAt
- route
- count
- p95Ms
- errorCount
If routes endpoint missing or response empty, disable button.
