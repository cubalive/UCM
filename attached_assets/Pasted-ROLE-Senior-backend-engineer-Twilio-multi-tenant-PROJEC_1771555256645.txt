PHASE 1 — CREATE A SINGLE TWILIO SERVICE LAYER
Implement a single module (example path):
- server/lib/sms/twilioClient.ts
- server/lib/sms/smsService.ts

Requirements:
- Initialize Twilio client once.
- Validate at startup:
  - TWILIO_ACCOUNT_SID starts with "AC"
  - TWILIO_AUTH_TOKEN non-empty
  - TWILIO_FROM_NUMBER in E.164 (+1xxxxxxxxxx)
- Provide function:
  sendSms({ companyId, to, body, purpose, tripId?, patientId?, driverId? })
- Enforce:
  - Normalize phone to E.164; reject if invalid.
  - Max body length guard (1600 ok, but warn above 320).
  - Prevent duplicates (idempotency key: companyId|purpose|tripId|to|YYYY-MM-DDTHH:mm bucket OR explicit idempotency key)
  - Per-number rate limit (configurable) to prevent spam loops
- Errors:
  - Map Twilio errors into { code, message, retryable }
  - Do not crash server on Twilio failures.
- Logging:
  - Log masked phone (last 2-4 digits only)
  - Log purpose + tenant + tripId
  - Never log message body if it contains PII (mask patient name/address).
