PHASE 3 — iOS location ingest endpoint (secure + throttled)
--------------------------------------------
Create/confirm endpoint: POST /api/driver/location
- Auth: DRIVER only via authMiddleware (Bearer token primary)
- Body (Zod):
  { lat:number, lng:number, speed?:number, heading?:number, accuracy?:number, ts?:number }
- Security:
  - Rate limit per driver userId (example: 1 request / 3 seconds, burst 5)
  - Reject if NOT connected AND driver has NO active (non-terminal) trip:
      return 403 { message:"Driver not connected" }
- Server-side throttle:
  - If last update < 3s ago AND movement < 20m => ignore write, return { ok:true, ignored:true }
- Persist:
  - Upsert latest location in existing store used by Live Map (reuse existing table or memory store)
  - Update drivers.lastSeenAt = now
- Realtime:
  - Broadcast updated location to websocket subscribers (dispatch/clinic views) using existing realtime module.

NOTE:
If you already have driverLocationIngest routes, do NOT duplicate logic.
Expose /api/driver/location as a stable alias that calls existing logic.
