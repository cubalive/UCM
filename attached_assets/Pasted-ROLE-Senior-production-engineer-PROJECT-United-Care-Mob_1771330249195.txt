PHASE 2 — ROLLUP ENGINE (DAILY METRICS)
========================================================

Create a server-side rollup job that computes daily_metrics_rollup from trips for a given city and date.

A) Endpoint (protected):
POST /api/intelligence/rollup/daily?city_id=...&date=YYYY-MM-DD
Auth: SUPER_ADMIN or DISPATCH

B) Logic:
- Pull trips with scheduled_pickup_at within date range for that city
- Compute:
  - trips_total
  - completed/cancelled/no_show
  - on_time_pickup_count (actual_pickup_at <= scheduled_pickup_at + 5 min)
  - late_pickup_count
  - avg_pickup_delay_minutes
  - gps_verified_count
  - pricing_missing_count (pricing_required AND price_cents is null for non-finalized)
  - invoices_missing_count (finalized AND invoice missing if invoices table exists)
  - revenue_cents (sum price_cents for completed)
  - est_cost_cents = (estimated miles * COST_PER_MILE_CENTS) + (duration minutes * COST_PER_MINUTE_CENTS)
  - margin_cents = revenue_cents - est_cost_cents
  - empty_miles, idle_minutes if available (else 0)
- Upsert unique keys for:
  - city-level row (clinic_id null, driver_id null)
  - clinic-level rows
  - driver-level rows

C) Constants (env configurable, defaults safe):
- COST_PER_MILE_CENTS default 65
- COST_PER_MINUTE_CENTS default 30

D) Idempotent:
- Re-running same date overwrites rollup for that date keys.

PHASE 3 — WEEKLY SCORES + RANKING (DPI/CRI/TRI/CostBleed)
========================================================

A) Endpoint (protected):
POST /api/intelligence/snapshots/weekly?city_id=...&weekStart=YYYY-MM-DD
Auth: SUPER_ADMIN or DISPATCH

B) Build weekly score snapshots using daily_metrics_rollup (7-day window)

Driver Performance Index (DPI, 0–100):
- punctuality = on_time_pickup / max(1, completed)
- completion  = completed / max(1, total)
- gps         = gps_verified / max(1, total_assigned_or_active)
- cost_eff    = 1 - clamp(empty_miles_ratio, 0, 1)
- idle_eff    = 1 - clamp(idle_minutes_ratio, 0, 1)
Score weights:
DPI = 35*punctuality + 20*completion + 20*gps + 15*cost_eff + 10*idle_eff

Clinic Reliability Index (CRI, 0–100) [clinic view only for their clinic]:
- no_show_control = 1 - (no_show / max(1,total))
- cancel_control  = 1 - (cancelled / max(1,total))
- punctuality_outcomes = on_time_pickup / max(1,completed)
CRI = 45*no_show_control + 25*cancel_control + 30*punctuality_outcomes

Transport Reliability Index (TRI, 0–100) per clinic and per city:
- on_time = on_time_pickup / max(1,completed)
- gps     = gps_verified / max(1,total)
- no_show_control = 1 - (no_show / max(1,total))
- billing_accuracy = 1 - (pricing_missing / max(1,total))   (proxy v1)
TRI = 40*on_time + 25*gps + 20*no_show_control + 15*billing_accuracy

Cost Bleed Score (0–100):
- margin_rate = margin / max(1,revenue)
- empty_ratio, idle_ratio
Score = clamp( 60*margin_rate + 20*(1-empty_ratio) + 20*(1-idle_ratio), 0, 100 )

C) Store:

- tri_scores rows for the same period with components jsonb

PHASE 4 — COST BLEED DETECTOR (ALERTS)
========================================================

A) Endpoint:
POST /api/intelligence/alerts/cost-bleed?city_id=...&date=YYYY-MM-DD
Auth: SUPER_ADMIN or DISPATCH

B) Rules (based on daily_metrics_rollup):
- IDLE: idle_minutes_ratio > 0.20 => YELLOW; >0.30 => RED
- EMPTY_MILES: empty_ratio > 0.35 => YELLOW; >0.45 => RED
- LOW_MARGIN: margin_rate < 0.15 => YELLOW; <0.05 => RED
- PRICING_GAPS: pricing_missing_count > 0 and trip pickup within next 2h => RED
- GPS_GAPS: gps_verified_rate < 0.90 => YELLOW; <0.80 => RED

C) Idempotent:
Do not duplicate same alert per (city_id, type, driver_id/clinic_id, date bucket) if OPEN.
