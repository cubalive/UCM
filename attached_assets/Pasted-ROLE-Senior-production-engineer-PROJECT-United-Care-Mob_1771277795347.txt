PHASE 1 — CORS MIDDLEWARE (STRICT)
---------------------------------------
Implement two CORS middlewares:

A) publicCors: apply ONLY to routes under /api/public/*
   - origin must be in allowedPublicOrigins
   - methods: GET,POST,OPTIONS
   - allowed headers: Content-Type, Authorization
   - credentials: false
   - respond to OPTIONS with 204

B) appCors: apply to ALL other /api/* routes
   - origin must be in allowedAppOrigins
   - methods: GET,POST,PUT,PATCH,DELETE,OPTIONS
   - allowed headers: Content-Type, Authorization
   - credentials: false (driver uses Bearer; do not rely on cookies)
   - respond to OPTIONS with 204

Do NOT use "*".

PHASE 2 — DRIVER AUTH (JWT BEARER, NO COOKIES REQUIRED)
---------------------------------------
1) Ensure /api/me accepts Authorization: Bearer <token> and returns:
   { ok:true, userId, role, companyId, email }
2) Ensure driver login endpoint returns a JWT:
   - If /api/auth/login-jwt exists, verify it is used by driver.
   - If missing, create POST /api/auth/login-jwt {email,password} -> {token,user}
3) Frontend driver must:
   - store token in localStorage key "ucm_driver_token"
   - attach Authorization header on every request via a single apiFetch wrapper
   - on 401: clear token and redirect to /login
