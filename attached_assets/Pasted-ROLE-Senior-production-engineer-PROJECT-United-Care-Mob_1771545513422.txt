========================================================
1.1) Verify DB schema for core entities (no changes yet):

1.3) Normalize address handling:

- normalizeAddress(input) -> trimmed string
- validateAddress(input) -> throws friendly error if empty
Store BOTH:
- formatted_address (string)
- lat/lng (numbers)
This prevents later geo-fence failures.

========================================================
5.1) Define geo-fence rules (implement once server-side):
- “Arrived at pickup” allowed only if driver location within X meters of pickup point
