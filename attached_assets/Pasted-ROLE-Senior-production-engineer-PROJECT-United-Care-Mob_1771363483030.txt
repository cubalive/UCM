PHASE 3 — STRIPE CONNECT (NO FUTURE REWORK)
------------------------------------------------------------
We will use Stripe Connect with destination charges:
- Platform creates Checkout Session
- Payment intent routes funds to connected account via transfer_data.destination
- Platform can optionally take fee via application_fee_amount (set to 0 for now but keep plumbing)

Secrets required:
- STRIPE_SECRET_KEY (platform)
- STRIPE_WEBHOOK_SECRET
- APP_URL (https://app.unitedcaremobility.com)

A) Connected account onboarding (SUPER_ADMIN only)
Endpoints:

1) POST /api/admin/companies/:companyId/stripe/connect/create
Auth: SUPER_ADMIN
- Creates Stripe connected account (type: 'express')
- Stores stripe_account_id in company_stripe_accounts
- Returns accountId

2) POST /api/admin/companies/:companyId/stripe/connect/onboarding-link
Auth: SUPER_ADMIN
- Creates account link for onboarding
- refresh_url: `${APP_URL}/admin/companies/${companyId}?stripe=refresh`
- return_url: `${APP_URL}/admin/companies/${companyId}?stripe=return`
- Returns { url }

3) GET /api/admin/companies/:companyId/stripe/connect/status
Auth: SUPER_ADMIN
- Fetch account from Stripe, update charges_enabled, payouts_enabled, details_submitted
- Set onboarding_status:
  ACTIVE if charges_enabled && payouts_enabled
  RESTRICTED otherwise
- Return status

UI:
- In company admin screen, show Stripe status and "Complete onboarding" button.

B) Clinic invoice payment (CLINIC_USER)
Endpoints:

1) GET /api/clinic/invoices
Auth: CLINIC_USER
Return invoices for actor.companyId + actor.clinicId

2) GET /api/clinic/invoices/:id
Auth: CLINIC_USER
Return detail if scoped else 404

3) POST /api/clinic/invoices/:id/pay
Auth: CLINIC_USER
Behavior:
- Fetch invoice by id AND scope to actor.companyId+clinicId (404 if not found)
- If status == PAID -> return { alreadyPaid: true, receipt_url, paid_at }
- Fetch company_stripe_accounts by actor.companyId
  - If missing or onboarding_status != ACTIVE:
    return 409 with message "Company billing not enabled yet"
- Create Stripe Checkout Session:
  - mode: "payment"
  - customer_email: clinic user email if available
  - line_items: [{ price_data: { currency, product_data:{name}, unit_amount }, quantity:1 }]
  - metadata: { invoice_id, company_id, clinic_id }
  - success_url: `${APP_URL}/invoices?paid=1&invoice=${invoice.id}`
  - cancel_url: `${APP_URL}/invoices?canceled=1&invoice=${invoice.id}`
  - payment_intent_data:
      - transfer_data.destination = stripe_account_id (CONNECTED ACCOUNT)
      - application_fee_amount = 0  (keep field so later you can charge fee without redesign)
      - metadata: same metadata
- Save stripe_checkout_session_id on invoice
- Return { url: session.url }

IMPORTANT:
- Do NOT mark invoice paid here.
- Only webhook marks paid.

C) Stripe webhook (public endpoint, signature required)
Endpoint:
POST /api/stripe/webhook

Rules:
- Verify signature with STRIPE_WEBHOOK_SECRET
- Persist stripe_event_id in stripe_webhook_events (unique) BEFORE processing
- If event already exists -> return 200 (idempotent)
Handle events:
1) checkout.session.completed
- Extract session.metadata.invoice_id
- Load invoice by id
- Verify invoice.company_id == session.metadata.company_id and clinic_id matches
- Fetch PaymentIntent and Charge to retrieve receipt_url if available
- Update invoice if not already PAID:
  - status = PAID
  - paid_at = now()
  - stripe_payment_intent_id = payment_intent
  - stripe_charge_id = latest_charge
  - receipt_url = charge.receipt_url or null
- Mark webhook event PROCESSED

Also optionally handle:
2) charge.succeeded / payment_intent.succeeded
- Only update if invoice found and not paid (same logic)

Failure handling:
- If mismatch -> mark event IGNORED and 200 (do not leak, do not retry forever)
- If exception -> mark FAILED, return 500 so Stripe retries
