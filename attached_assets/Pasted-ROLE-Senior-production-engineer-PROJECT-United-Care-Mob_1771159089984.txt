SLIDE-UP DRAWER (BOTTOM SHEET MENU)
------------------------------------------------------------
Drawer opens upward and contains sections in this order:

SECTION 1 — DRIVER PROFILE
- Driver photo/avatar
- Name
- Vehicle (optional)
- Current status (Online/Offline)
- Small quick link: "Edit Profile" (if allowed)

SECTION 2 — TRIPS
Tabs inside the drawer:
- Scheduled Trips (today + upcoming)
- Trip History (completed/cancelled)
Both lists open Trip Details.

SECTION 3 — MY SCHEDULE
Show driver schedule (week view)
- Read-only by default
- Shows assigned working days/hours
- If Sunday/extra shifts exist, show clearly

SECTION 4 — REQUEST DAY CHANGE
A form to request a schedule change:
- Pick date
- Pick type: swap/cover/unavailable
- Notes
- Submit

Backend behavior:
- Create a schedule_change_request record
- Notify dispatch by email
- Dispatch approves/denies in dispatcher UI
- Driver receives email when approved/denied
- Driver sees status in the drawer (Pending/Approved/Denied)

SECTION 5 — METRICS
Show driver metrics summary:
- Completion rate
- On-time rate
- No-show handling (if tracked)
- Weekly totals
Pull from existing dispatcher metrics source (do NOT invent; reuse same calculation).

SECTION 6 — WEEKLY BONUS (ONLY WHEN ACTIVE)
This section is hidden unless a bonus program is active.
When active:
- Show progress % toward qualification
- Show a progress bar with color states:
  - Red (below threshold)
  - Yellow (near threshold)
  - Green (qualifies)
- Show exact requirement text (e.g., "Maintain ≥X% on-time and ≥Y trips")

Rules:
- Bonus configuration comes from dispatcher/admin settings (existing or add minimal)
- Driver view is read-only

SECTION 7 — LOG OUT / GO OFFLINE (AT BOTTOM)
At the very bottom of drawer:
- "Go Offline" button (or "Disconnect")
This must be separate and visually distinct.
Main screen bottom strip still includes Online/Offline toggle; drawer also includes it at bottom for convenience.
