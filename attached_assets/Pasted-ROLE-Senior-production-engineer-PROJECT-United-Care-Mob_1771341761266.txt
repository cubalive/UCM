2C) DAILY ROLLUP ENDPOINT
------------------------------------
POST /api/intelligence/rollup/daily?city_id=...&date=YYYY-MM-DD
Auth: SUPER_ADMIN or DISPATCH

Logic:
1) Validate date and city_id
2) Enforce city scoping for DISPATCH
3) Query trips where scheduled_pickup_at in [date 00:00, date+1 00:00) AND city_id = given
4) Derive metrics:
- trips_total
- trips_completed (status indicates completed OR actual_dropoff_at not null OR finalized)
- trips_cancelled (cancelled=true OR status in cancelled states)
- trips_no_show (no_show=true)
- on_time_pickup_count:
  actual_pickup_at <= scheduled_pickup_at + 5 minutes
- late_pickup_count = completed - on_time
- avg_pickup_delay_minutes:
  avg(max(0, minutes(actual_pickup_at - scheduled_pickup_at))) for trips with both times
- gps_verified_count = count(gps_verified=true)
- pricing_missing_count:
  pricing_required=true AND price_cents is null AND NOT cancelled AND NOT no_show
- invoices_missing_count:
  if invoices table exists in schema, count finalized trips missing invoice; else 0
- revenue_cents = sum(price_cents) for completed, else 0
- est_cost_cents:
  use existing trip distance/duration fields if present (preferred)
  else safe fallback:
    - miles=0, minutes=0 so cost=0 (do not guess with external APIs)
- margin_cents = revenue - est_cost
- empty_miles, idle_minutes:
  use existing telemetry fields if present; else 0

5) Upsert daily_metrics_rollup rows for:
- city-level (clinic_id null, driver_id null)
- clinic-level (clinic_id grouped, driver_id null)
- driver-level (driver_id grouped, clinic_id preserved if present)

Idempotent:
- Upsert by unique key (metric_date, city_id, clinic_id, driver_id)
- Overwrite values on rerun

------------------------------------
2D) WEEKLY SNAPSHOTS ENDPOINT
------------------------------------
POST /api/intelligence/snapshots/weekly?city_id=...&weekStart=YYYY-MM-DD
Auth: SUPER_ADMIN or DISPATCH

Logic:
1) week_end = weekStart + 7 days
2) Load 7 days from daily_metrics_rollup for the city in range
3) Aggregate by:
- city-level
- clinic-level
- driver-level
4) Compute scores (0–100) with clamp helper:

Driver Performance Index (DPI):
punctuality = on_time_pickup_count / max(1, trips_completed)
completion  = trips_completed / max(1, trips_total)
gps         = gps_verified_count / max(1, trips_total)
empty_ratio = empty_miles / max(1, (empty_miles + paid_miles))  (if paid_miles unavailable, treat empty_ratio=0)
idle_ratio  = idle_minutes / max(1, (idle_minutes + active_minutes)) (if unavailable, treat idle_ratio=0)
cost_eff    = 1 - clamp(empty_ratio,0,1)
idle_eff    = 1 - clamp(idle_ratio,0,1)
DPI = 35*punctuality + 20*completion + 20*gps + 15*cost_eff + 10*idle_eff

Clinic Reliability Index (CRI) (store but only clinic can view their own):
no_show_control = 1 - (trips_no_show / max(1,trips_total))
cancel_control  = 1 - (trips_cancelled / max(1,trips_total))
punctuality_outcomes = on_time_pickup_count / max(1,trips_completed)
CRI = 45*no_show_control + 25*cancel_control + 30*punctuality_outcomes

Transport Reliability Index (TRI):
billing_accuracy = 1 - (pricing_missing_count / max(1,trips_total))
TRI = 40*on_time + 25*gps + 20*no_show_control + 15*billing_accuracy

Cost Bleed Score:
margin_rate = margin_cents / max(1,revenue_cents)
Score = clamp(60*margin_rate + 20*(1-empty_ratio) + 20*(1-idle_ratio), 0, 100)

5) Upsert weekly_score_snapshots for each grouping (unique week_start, city, clinic, driver)
6) Upsert tri_scores (city + each clinic) with components json:
{ on_time, gps, no_show_control, billing_accuracy, inputs: {counts...} }
