GLOBAL FIX: ADDRESS AUTOCOMPLETE EVERYWHERE

PROBLEM  
Address fields across the system do NOT suggest addresses.  
This causes wrong pickups, wrong billing, and confusion for clinics and dispatch.  

REQUIREMENT  
Every place where an address is typed must use ONE standardized autocomplete system.

This includes:  
- Add Patient (clinic + dispatch)  
- Edit Patient  
- Create Trip (pickup + dropoff)  
- Edit Trip  
- Add Clinic  
- Private booking form  
- Any quote or manual trip creation screen  

--------------------------------------------------
PHASE 1 — CREATE GLOBAL AddressAutocomplete COMPONENT
--------------------------------------------------
Create a reusable component: AddressAutocomplete

Behavior:
- Shows suggestions while typing (debounce 300ms)
- Uses backend endpoint
- User must select from suggestions
- After selection store:
  formatted_address  
  lat  
  lng  
  place_id  
  optional unit/apt  

UI must:
- Work inside modals
- Work on mobile
- Large tap targets
- Dropdown visible above modal
