STEP 3 — Dispatch Window Engine (scheduler)
Create a scheduler that runs every 60s:
For each assigned trip in next 24h that is not terminal:
1) Determine driver current location (latest ping). If missing, fallback to last known location.
2) Compute ETA driver->pickup using existing Google Routes/Directions integration.
3) Compute buffer_min based on service type:
   - ambulatory: 10
   - wheelchair: 15
4) notify_lead_min = 5
5) dispatch_at = scheduled_pickup_at - (eta_to_pickup_min + buffer_min)
6) notify_at = dispatch_at - notify_lead_min
7) Save dispatch_at/notify_at if changed.

Notification rules (idempotent):
- If now >= notify_at and dispatch_stage='NONE': send push/SMS/in-app notification “Trip in 5 min — get ready” and set dispatch_stage='NOTIFIED'
- If now >= dispatch_at and dispatch_stage IN ('NONE','NOTIFIED'):
   a) set driver availability_status='BUSY'
   b) transition trip status to EN_ROUTE_TO_PICKUP (via existing transition function)
   c) compute and store polyline + distance + duration for driver->pickup route
   d) set dispatch_stage='DISPATCHED'
   e) notify driver “Go to pickup now”

STEP 4 — Driver Acceptance Logic
