PHASE 1 — ALERT DELIVERY RULES (ANTI-SPAM)
Create table if missing:
ops_alert_log
- id uuid pk
- city_id text
- date date
- alert_fingerprint text unique
- overall text
- critical_codes text[] or jsonb
- sent_at timestamptz default now()
- sent_to text
- provider_sid text null
- error text null

Define fingerprint:
hash(city_id + date + overall + sorted(list_of_critical_codes) + counts_bucket)
Counts bucket can be simplified (just codes) to avoid spam from tiny count changes.

Send SMS only when:
A) overall becomes "red" and no alert sent in last 30 minutes for same city/date
OR
B) a new CRITICAL alert code appears that was not in the last sent fingerprint

Cooldown:
- Minimum 30 minutes between CRITICAL SMS per city (unless a brand-new CRITICAL code appears)

PHASE 2 — MESSAGE FORMAT (SHORT + ACTIONABLE)
SMS format:
"UCM OPS ALERT ({CITY}) RED:
{CODE1}:{COUNT1}, {CODE2}:{COUNT2}
Open: {OPS_URL}"

Where:
OPS_URL = APP_PUBLIC_URL + "/ops-health?city={city_id}"

If message too long, include top 3 critical alerts only.
