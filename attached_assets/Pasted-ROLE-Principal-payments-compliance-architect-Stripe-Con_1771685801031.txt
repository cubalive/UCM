SECTION 2 — STRIPE CONNECT PAYMENT ENGINE (DIRECT PAY)
========================================================

A) Connected account onboarding
- SUPER_ADMIN endpoint to create Express account if missing.
- Create AccountLink for onboarding; store status flags.
- Add SUPER_ADMIN “Companies → Stripe Status” screen with:
  pending/active/restricted + “Resume onboarding” button.

B) PaymentIntent creation (weekly billing period pay)
Endpoint: POST /api/clinic/billing/:billingPeriodId/pay
Rules:
- Verify requester is CLINIC for that clinic_id or SUPER_ADMIN.
- Verify company has stripe_connected_account_id and charges_enabled.
- Create or fetch clinic stripe_customer_id.
- Ensure a default payment method is attached and set.
- Create PaymentIntent:
    amount = total_cents
    currency = usd
    customer = clinic.stripe_customer_id
    payment_method = clinic.stripe_default_payment_method_id (if exists)
    off_session = optional (if saved and permitted)
    confirm = true when off_session, else return client_secret for confirmation
    transfer_data.destination = company.stripe_connected_account_id
    application_fee_amount = configurable per company/clinic (fee rules)
    metadata includes billing_period_id, clinic_id, company_id, period_start/end
- Persist payments row in DB.
- Write audit_events + ledger_entries.

C) Webhooks (idempotent)
Handle:
- payment_intent.succeeded
- payment_intent.payment_failed
- charge.refunded
- charge.dispute.created / updated / closed
- transfer.created / paid / reversed (if used)
- account.updated (connect status)
Requirements:
- Idempotency table or store processed event ids.
- Update billing_periods status accordingly.
- Insert audit_events and ledger_entries.
- For reconciliation: fetch balance transactions and persist payout_reconciliation.

========================================================

Implement a scheduler:
