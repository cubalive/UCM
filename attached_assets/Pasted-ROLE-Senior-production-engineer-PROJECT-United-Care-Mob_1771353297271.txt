TASK (SINGLE TASK ONLY):
Implement ARCHIVE / UNARCHIVE for Clinics, Drivers, Patients with strict RBAC + tenant scoping.
Admin (SUPER_ADMIN) and Company roles (DISPATCH / CLINIC_ADMIN if exists) can archive/unarchive within their tenant.
No hard deletes. No DB reset. Non-breaking.

DEFINITION:
Archive = make record inactive AND set deletedAt timestamp (soft delete) WITHOUT removing data.
Unarchive = set active=true AND deletedAt=NULL.

SCOPE:
Entities:
- clinics
- drivers
- patients

Roles:
- SUPER_ADMIN: can archive/unarchive ANY tenant.
- DISPATCH: can archive/unarchive ONLY within actor.companyId and ONLY within actor.allowedCityIds (if city scoped).
- CLINIC_USER: cannot archive/unarchive (read-only for their clinic).
- DRIVER: cannot archive/unarchive.

HARD REQUIREMENTS:
1) Server-side enforcement only (client cannot bypass).
2) Tenant isolation: ALWAYS filter by actor.companyId unless SUPER_ADMIN.
3) City isolation for DISPATCH if userCityAccess exists: validate entity.cityId is in allowed list.
4) Prevent archiving if entity has ACTIVE trips:
   - clinics: block if has active trips (SCHEDULED/ASSIGNED/IN_PROGRESS)
   - drivers: block if has active trips
   - patients: block if has active trips
5) Audit log entry for every archive/unarchive:
   action, entity_type, entity_id, actor_user_id, actor_role, company_id, city_id, before/after snapshot summary.

Rules inside each method:
1) Load entity by id with scoping:

   - else: where id AND companyId=actor.companyId (if column exists) AND (cityId in allowedCityIds for DISPATCH if city scoped)

3) Active trip guard:
   - Query trips where entity matches AND status in ('SCHEDULED','ASSIGNED','IN_PROGRESS') AND deletedAt IS NULL
   - If exists -> throw 409 with message "Cannot archive: active trips exist"
4) Update entity:
   - archive: set active=false, deletedAt=now()
   - unarchive: set active=true, deletedAt=null
5) Insert auditLog row:
