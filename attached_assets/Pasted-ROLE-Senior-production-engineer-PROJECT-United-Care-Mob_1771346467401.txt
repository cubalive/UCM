PHASE 1 — DEFINE THE CLINIC-VISIBLE INDEX SET (NO INTERNAL PROFIT)
------------------------------------------------------------
Expose THESE 10 indices to clinics (clinic-safe, no profit/leakage dollars unless you decide later):

1) Transport Reliability Index (TRI Nevada™) 0–100
2) Clinic Trust Score (CTS) 0–100
3) Treatment Continuity Index (TCI) %   (completed clinic trips ÷ scheduled clinic trips)
4) Chair Protection Index %            (on-time arrivals to clinic ÷ clinic arrivals)
5) Same-Day Recovery Index %           (recovered incidents ÷ incidents) [if incidents exist; else omit + reweight]
6) Return Reliability Index %          (on-time returns ÷ returns)
7) Audit Readiness Index %             (trips with full proof: timestamps completeness + photo/GPS proof)
8) Medical Punctuality Index %         (arrived within clinic buffer threshold)
9) Late Risk Predictor (Quarter summary) (red/yellow/green counts + top reasons)
10) Service Stability Index 0–100      (completed ÷ assigned + penalties) [clinic-scoped variant]

IMPORTANT:
- Each index must show: value, trend vs last quarter, and the counts behind it.
- If any sub-component data not available, omit that component and normalize weights (no blanks).

PHASE 2 — QUARTER MODEL + SNAPSHOTS (REQUIRED FOR CONSISTENCY)
------------------------------------------------------------
We must freeze quarterly results so they don’t change later.

Create tables (minimal):

clinic_quarterly_reports
- id uuid pk
- clinic_id uuid
- state text
- city text
- quarter_key text  -- e.g. "2026-Q1"

UNIQUE(report_id, metric_key)

Ranking snapshots:
quarterly_rankings
- id uuid pk

- state text nullable
- city text nullable
- metric_key text  -- primarily TRI and CTS

quarterly_ranking_entries
- id uuid pk

UNIQUE(ranking_id, clinic_id)

PHASE 3 — COMPUTATION JOB (EVERY QUARTER)
------------------------------------------------------------
Implement quarterly compute job:
- Determine current quarter boundaries.
- For each clinic:
  - compute 10 metrics for period_start..period_end
  - compute trend vs prior quarter (using previous snapshot)
  - store in clinic_quarterly_reports + clinic_quarterly_report_metrics
- Compute rankings:
  - City ranking (within city)
  - State ranking (within state)
  - National ranking (all clinics)
Ranking primary metric:
- TRI Nevada™ (default)
Secondary tie-breaker:
- CTS
Store top N and each clinic rank.

No external APIs.
