SECTION D — AUTHENTICATION + SESSION SECURITY
========================================================

D1) Identify auth model:
- Supabase JWT vs internal JWT vs session cookies

D2) Verify auth middleware exists server-side:

- requireRole(...roles)

D3) Verify token storage & transport:
- If cookies: HttpOnly, Secure, SameSite
- If bearer token: verify standard Authorization header flow

PASS criteria:

- Role-based routes reject wrong roles (403)
- No user-controlled clinic_id / city_id overrides for restricted roles

Output:
- AUTH_MODEL: ...

- COOKIE/Bearer security notes
