PHASE 1 — AUTO-ASSIGN INTELIGENTE V2
=========================================================
GOAL:
Fully automated score-based driver assignment with escalation, plus interactive UI to understand and control it.

STEP 1 — DATABASE (Idempotent Migration)
If companies table exists, add:

auto_assign_enabled BOOLEAN DEFAULT true
auto_assign_offer_timeout_seconds INTEGER DEFAULT 120
auto_assign_max_rounds INTEGER DEFAULT 6
auto_assign_max_distance_meters INTEGER DEFAULT 20000
auto_assign_weight_distance INTEGER DEFAULT 45
auto_assign_weight_reliability INTEGER DEFAULT 25
auto_assign_weight_load INTEGER DEFAULT 20
auto_assign_weight_fatigue INTEGER DEFAULT 10

If columns exist → do nothing.

Also add to trips table (if not exists):
auto_assign_status TEXT DEFAULT 'IDLE'  -- IDLE|RUNNING|SUCCESS|FAILED|PAUSED
auto_assign_last_run_at TIMESTAMPTZ
auto_assign_failure_reason TEXT
auto_assign_selected_driver_id UUID NULL

Also add table (if not exists) auto_assign_runs:
id UUID PK
trip_id UUID
company_id UUID
city_id UUID
started_at TIMESTAMPTZ
ended_at TIMESTAMPTZ
result TEXT  -- SUCCESS|FAILED|CANCELLED
selected_driver_id UUID NULL
reason TEXT NULL

Also add table (if not exists) auto_assign_run_candidates:
id UUID PK
run_id UUID
driver_id UUID
distance_meters INTEGER
reliability_score NUMERIC
load_score NUMERIC
fatigue_score NUMERIC
final_score NUMERIC
rank INTEGER
eligible BOOLEAN
ineligible_reason TEXT NULL

(Keep it minimal but traceable.)

STEP 2 — DRIVER CANDIDATE POOL
When trip becomes READY_TO_ASSIGN:
Select drivers:
- same company_id + same city_id
- active shift
- no conflicting trip
- within auto_assign_max_distance_meters
- last GPS < 120 seconds

STEP 3 — SCORING FORMULA (0–100 normalized)
DistanceScore = 1 - (distance_to_pickup / max_distance)
ReliabilityScore = driver_on_time_rate (0–1)
LoadScore = 1 - (active_trips / max_allowed_trips)
FatigueScore = 1 - (hours_worked_today / shift_limit)

FinalScore =
(w_distance*DistanceScore) +
(w_reliability*ReliabilityScore) +
(w_load*LoadScore) +
(w_fatigue*FatigueScore)

Normalize weights from company config.
Store all candidate scores in auto_assign_run_candidates.

STEP 4 — OFFER FLOW
Offer to top driver:
- create offer row
- TTL = auto_assign_offer_timeout_seconds
- accept → assign trip + mark run SUCCESS
- decline/timeout → next driver
- max rounds = auto_assign_max_rounds

If exhausted → set trip AUTO_ASSIGN_FAILED + show clickable “Why failed” + notify dispatcher.

Add per-trip in-memory cooldown (30s) to prevent loops.

STEP 5 — UI/INTERACTION FOR AUTO-ASSIGN
Create an “Auto-Assign” panel with:
- Status chip (RUNNING/SUCCESS/FAILED/OFF)
- “Explain” link: shows last run summary
- “View candidates” link: opens ranked list (click driver to open driver profile)
- “Retry” button (if failed)
- “Override” button (assign manually)
- “Settings” link (company config)

Trip detail page:
- Show “Auto-Assign Timeline” (click any event to see payload)
- Show selected driver + score breakdown

PHASE 2 — ETA VARIANCE ESCALATION ENGINE
=========================================================
GOAL:
Detect lateness early + interactive escalation center.

STEP 1 — Trip fields (idempotent)
original_eta_seconds INTEGER NULL
eta_last_checked_at TIMESTAMPTZ NULL
eta_variance_seconds INTEGER NULL
eta_escalation_level TEXT DEFAULT 'NONE' -- NONE|WARN|CLINIC|DISPATCH

STEP 2 — Recalc on GPS (throttle 60s)
Recalculate ETA using stored route_polyline (DO NOT call Directions).
variance = current_eta - original_eta

Thresholds:
> 300s → WARN
> 600s → CLINIC
> 900s → DISPATCH

No duplicate escalation per threshold; store last escalation timestamp.

STEP 3 — UI/INTERACTION
Create “ETA Escalations” page:
- Filter pills: WARN / CLINIC / DISPATCH
- Each row clickable → trip detail with “why late” breakdown:
   * driver distance trend
   * stop time / wait time
   * last GPS timestamp
   * original vs current ETA
- Quick actions: “mute 30 min”, “reassign driver”, “notify clinic”
