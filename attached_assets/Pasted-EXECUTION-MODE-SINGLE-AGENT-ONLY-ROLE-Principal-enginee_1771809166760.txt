STEP 2 — Auto-assign engine (core)
Create server/lib/autoAssignEngine.ts and a scheduler (every 30-60s) plus event-hook:
Trigger auto-assign on:
- trip created by clinic and marked APPROVED/CONFIRMED
- any time trip is edited and still not terminal
- periodic retry for PENDING/FAILED

Auto-assign algorithm:
Hard filters:
- driver in same tenant/company/city scope
- driver category supports service type (ambulatory/wheelchair)
- driver availability_status != OFFLINE
- driver has no time-conflict with existing assigned trips

Priority order:
1) Frequent patient preference: if patient.is_frequent=true AND patient.preferred_driver_id exists:
   if feasible -> assign that driver; set auto_assign_reason='preferred_driver'
2) Otherwise score all candidates with:
   score = 0
   + 1000 if driver matches preferred_driver_id snapshot
   + 200 if driver clinic affinity high (based on past on-time for that clinic)
   + 0..300 based on lower ETA driver->pickup
   + 0..150 based on lower deadhead distance
   + 0..200 based on driver performance (punctuality, cancel rate, compliance)
   - big penalty if tracking_status != OK
Pick highest score.

Time conflict check:
For each driver’s existing assigned trips on same day:
- compute dispatch window for those trips (or approximate)
- if overlap with candidate trip dispatch window -> conflict -> reject candidate

On success:
- set trip.driver_id
- set trip.auto_assign_status='ASSIGNED'
- set trip.auto_assign_reason with winner reason
- do NOT mark driver BUSY unless now>=dispatch_at (handled later)

On failure:
- set auto_assign_status='FAILED' + reason ('no_candidates', 'all_conflicted', etc)
- emit websocket event to dispatcher/admin for manual handling

STEP 3 — Compute ETA + dispatch window fields (saved on trip)
For any assigned trip:
- compute eta_driver_to_pickup_min via Google Routes (driver last known location -> pickup)
- set buffer:
  wheelchair=15, ambulatory=10 (or use service_buffer_min per trip)
- notify_lead=5
- dispatch_at = scheduled_pickup_at - (eta_driver_to_pickup_min + buffer)
- notify_at = dispatch_at - 5 minutes
Persist these values.
